## 1. 背景

スターバックスのドリンクレシピは、スチーム温度・抽出ショット数・シロップポンプ数など、工程ごとに細かいパラメータが存在し、かつサイズ（S/T/G/V）やカスタム（ミルク変更・シロップ追加・ショット変更など）によって多数のバリエーションが生まれる。現在、スタッフが参照できる体系的な自主学習用ツールは存在せず、新人アルバイトスタッフが正確にレシピを習得するためには、先輩スタッフへの質問や現場での試行錯誤に頼らざるを得ない状況が続いている。

本アプリは、この課題を解決するためにスターバックスのアルバイトスタッフを対象に設計する自主学習型モバイルアプリである。レシピの参照・実践練習・振り返り記録を一元化し、スキル習得サイクルを加速させることを目的とする。

## 2. 課題

| # | 課題 | 詳細 |
|---|------|------|
| P1 | 自主学習環境の欠如 | 体系的なレシピ書や研修アプリが存在しない。新人スタッフが自宅で自主学習できる手段がほぼない。 |
| P2 | レシピの複雑性 | 46品以上のドリンクに対し、サイズ・カスタムの組み合わせが膨大で、全パターンを記憶するのが困難。 |
| P3 | 弱点の把握困難 | 自分がどのドリンクを苦手としているかを客観的に把握する仕組みがない。 |
| P4 | 振り返りの欠如 | シフト後の振り返りを記録する習慣づけが難しく、学習の定着が遅れる。 |
| P5 | マルチタスク対応力の不足 | 複数オーダーを同時にこなす際の優先順位・作業順の判断力が育ちにくい。 |

## 3. 対象ユーザー

### 3.1 主要ユーザー

| 項目 | 内容 |
|------|------|
| 主なユーザー | スターバックスでアルバイト勤務している学生（18〜25歳程度） |
| 利用シーン | 自宅学習・通学中・出勤前の予習・シフト後の振り返り |
| 技術リテラシー | スマートフォン操作に慣れており、アプリの基本操作に不安がない |
| 勤務経験 | 入社直後〜6ヶ月程度の新人〜中堅層を主な対象とする |

### 3.2 周辺ステークホルダー

| ステークホルダー | 関係性 |
|----------------|--------|
| 先輩スタッフ・店長 | 新人の学習進捗を間接的に受益するが、アプリへの直接アクセスは現時点では想定しない |
| アプリ開発者 | レシピデータのメンテナンス・バグ修正などを担う |

## 4. 提供価値

| 価値 | 説明 |
|------|------|
| いつでも・どこでも自主学習 | オフライン対応により、ネットワーク環境がなくてもアプリの全機能を利用できる。通学中や休憩時間の隙間学習が可能。 |
| 実践的な練習形式 | ランダムなオーダーを想定した手順並び替え問題により、実際の店舗業務に近い感覚で練習できる。 |
| 弱点の可視化 | 間違い問題リストと習得度ステータスにより、自分の苦手なドリンクや工程を客観的に把握できる。 |
| 振り返り習慣の醸成 | 日次振り返り入力機能により、シフト後の気づきを記録し、学習定着を促す。 |
| モチベーション維持 | ゲーミフィケーション（連続正解ストリーク・習得度バッジ・総合スコア）により、継続学習へのモチベーションを高める。 |
| ユーザー主体のカスタマイズ | 期間限定メニューをユーザー自身が登録できるため、常に最新のメニューに対応した学習が可能。 |

## 5. 成功指標（KPI）

### 5.1 ユーザーエンゲージメント

| KPI | 目標値 | 測定方法 |
|-----|--------|---------|
| 週次アクティブ率 | 週3回以上の利用率 70% 以上 | 練習セッション起動ログ |
| 連続利用日数 | 平均7日以上の継続利用 | アプリ起動ログ |
| 振り返り記録率 | 利用日の50%以上で振り返り記録 | 振り返りレコード数 / 利用日数 |

### 5.2 学習成果

| KPI | 目標値 | 測定方法 |
|-----|--------|---------|
| ドリンク習得率 | 30日後に全46品中80%以上「習得済み」 | USER_PROGRESS テーブル |
| 練習セッション正解率 | 利用開始30日後に平均80%以上 | PRACTICE_SESSION.correct_count / 10 |
| 間違い問題消化率 | 要復習リストを1週間以内に50%以上消化 | WRONG_ANSWER.resolved_flag |

### 5.3 品質指標

| KPI | 目標値 |
|-----|--------|
| アプリクラッシュ率 | 0.1% 以下 |
| 画面遷移速度 | 300ms 以内 |
| アプリ起動時間 | 2秒以内（コールドスタート） |

## 6. スコープ

### 6.1 やること（In Scope）

- **M1: レシピ管理モジュール**
  - システム組込み46品のレシピ閲覧（一覧・詳細）
  - カテゴリ・サブカテゴリ・テキストによるフィルタリング・検索
  - サイズ別分量（S/T/G/V）の表示
  - カスタムバリエーション（ミルク変更・シロップ追加等）の切替表示
  - ユーザーによる期間限定レシピの登録・編集・削除

- **M2: 実践練習モジュール**
  - 難易度・カテゴリを選択したランダムオーダー生成（10問/セッション）
  - 手順並び替え問題（ドラッグ&ドロップ）
  - 分量クイズ問題（オプション）
  - 10オーダーの作業順並び替え練習
  - セッション終了後の採点・フィードバック表示

- **M3: 振り返り・学習記録モジュール**
  - バイト振り返りの入力・一覧・詳細表示（シフト日・うまくできたこと・ミス・フィードバック・次の復習事項・手応えの記録）
  - 間違い問題リストの確認・再練習
  - 学習進捗ダッシュボード（習得率・要復習件数・苦手カテゴリ）

- **非機能**
  - iOS / Android 対応（React Native / Expo）
  - オフライン完全対応（SQLite 端末内保存）
  - ゲーミフィケーション（連続正解・習得度表示・スコア）

### 6.2 やらないこと（Out of Scope）

| 項目 | 理由 |
|------|------|
| ユーザーアカウント機能 / サーバー同期 | 初期フェーズではシングルデバイス利用を想定。複数デバイス同期は今後の検討事項。 |
| Web ブラウザ対応 | モバイルアプリ優先。Web 版は開発ロードマップに含まない。 |
| 管理者ダッシュボード（スタッフ管理） | 店長・マネージャー向けの進捗一覧画面は対象外。 |
| レシピ画像添付 | 初期フェーズでは画像登録機能を持たない。テキストベースの登録のみ。 |
| Push通知 | 練習リマインダー通知は今後の検討事項。 |
| ソーシャル機能 | 先輩コメント・レシピ共有などのソーシャル要素は対象外。 |
| 決済・課金機能 | 本アプリは個人開発の学習ツールであり、課金は対象外。 |

## 7. 制約・前提

### 7.1 技術的制約

| 制約 | 内容 |
|------|------|
| プラットフォーム | iOS / Android のみ。Web は非対応。 |
| ネットワーク | オフライン完全対応必須。全データは端末内 SQLite に保存。 |
| フレームワーク | React Native / Expo を使用。TypeScript 必須。 |
| ドラッグ&ドロップ | react-native-reanimated + gesture-handler を使用。 |

### 7.2 ビジネス上の制約

| 制約 | 内容 |
|------|------|
| レシピデータ | システム組込みレシピは開発チームが入力する。ユーザーは削除・改変不可。 |
| データ保護 | 端末内保存のため、端末紛失・初期化でデータは消失する（バックアップは対象外）。 |
| 著作権 | レシピデータはスターバックスジャパンの営業情報に基づく。商用利用・外部公開は禁止。 |

### 7.3 前提条件

| 前提 | 内容 |
|------|------|
| ユーザー端末 | iPhone / Android スマートフォンを保有している |
| 操作スキル | スマートフォンの基本操作（タップ・スクロール・ドラッグ）ができる |
| 単一ユーザー | 1台の端末を1人のユーザーが使用する前提（マルチアカウント非対応） |

## 8. 用語集

| 用語 | 説明 |
|------|------|
| M1 | レシピ管理モジュール（Module 1）の略称 |
| M2 | 実践練習モジュール（Module 2）の略称 |
| M3 | 振り返り・学習記録モジュール（Module 3）の略称 |
| builtin | システムが最初から組み込んでいるレシピを指す。削除・改変不可。 |
| user | ユーザーが自分で登録した期間限定レシピを指す。編集・削除可。 |
| S / T / G / V | スターバックスのカップサイズ。S=ショート, T=トール, G=グランデ, V=ベンティ。 |
| カスタム | ミルク変更・シロップ追加・ショット変更など、標準レシピからの変更オプション。 |
| ストリーク | 連続正解回数のこと。ゲーミフィケーション要素として表示される。 |
| 習得済み | 直近5回の練習で全問正解した状態を指す習得ステータス。 |
| 要復習 | 練習で間違えた問題に付与されるフラグ。正解するまでリストに残る。 |
| セッション | 1回の練習につき10問オーダーをこなす単位。PRACTICE_SESSION として記録される。 |
| フラペチーノ | ブレンドタイプのドリンクカテゴリ。バイタミックス等の特殊機材を使用する。 |
| バイタミックス | フラペチーノをブレンドするための専用ミキサー。特殊機材として扱う。 |
| XH ボタン | エクストラホット設定ボタン。非乳性ミルク（豆乳・オーツミルク等）使用時に必須。 |
| スリーブ | 熱いカップに巻く保護用の紙製カバー。スリーブ要否フラグで管理される。 |
| IPO | Input / Process / Output の略。システムの処理フローを整理する際に使用する。 |
