# レシピ手順構造化ノート（提供テキスト反映版）

**作成日**: 2026-02-22  
**元データ**: ユーザー提供の生テキスト（本ファイルに準拠）

本ノートは、受領したレシピ手順テキストを以下の3形式に整理した。

1. **Markdownサマリー（本ファイル）** — コード・カテゴリ・主要ステップを人が読める形で一覧化  
2. **画面表示用JSON** — `src/constants/recipes-ui.json` に保存。UI表示・シード共用の最小構造（name, shortCode, category, needsSleeve, steps）  
3. **SQLiteシード** — `src/db/seed.ts` で JSON を読み込み、アプリ起動時に投入

> 注: 量が多いため、分量ポンプ数などはステップ説明のテキストに含めている。必要に応じて `ingredients` 拡張を今後追加可能。

---

## 収録フィールド

- **code**: レジ略称（例: MIS, L, A）
- **name**: ドリンク正式名
- **category**: `hot` / `ice` / `frappuccino` / `seasonal`
- **needsSleeve**: スリーブ要否（true/false）※熱いカップで明記されているもののみ true
- **steps**: 番号付きの手順テキスト
