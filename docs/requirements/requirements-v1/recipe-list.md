# 画面要件定義 v1: レシピ一覧（M1-01）

**作成日**: 2026-02-22

---

## 1. 機能要件

| # | 機能 | 説明 |
|---|------|------|
| F01 | カテゴリタブ切替 | ホット / アイス / フラペチーノ / 期間限定 / すべて の5タブで絞り込み表示する |
| F02 | サブカテゴリフィルター | 横スクロールチップで「エスプレッソ系」「ティー系」等のサブカテゴリ絞り込みができる |
| F03 | テキスト検索 | ドリンク名（日本語）・略称コードによるインクリメンタルサーチを実装する |
| F04 | ドリンクカード表示 | ドリンク名・略称コード・カテゴリバッジ・スリーブアイコン・習得度スターをカードで表示する |
| F05 | 習得度スター表示 | USER_PROGRESS の習得ステータスに応じて ☆（未学習）/ ★（学習中）/ ★★（習得済み）を表示 |
| F06 | ユーザー登録バッジ | recipe_type = 'user' のレシピカードには「ユーザー登録」バッジを右上に表示する |
| F07 | 新規登録FAB | 右下の＋ボタンでユーザーレシピ登録画面（M1-04）へ遷移する |
| F08 | カード長押しメニュー | ユーザー作成レシピのカードを長押しすると「編集」「削除」のコンテキストメニューを表示する |
| F09 | 削除確認ダイアログ | 削除選択時に「このレシピを削除しますか？」確認ダイアログを表示する |

## 2. 非機能要件

| # | 項目 | 要件 |
|---|------|------|
| NF01 | 表示速度 | 全ドリンク一覧の初期表示を500ms以内に完了すること |
| NF02 | スクロール性能 | FlatList/FlashListを使用し、スクロール時のフレームドロップを防ぐこと（60fps維持） |
| NF03 | 検索レスポンス | 検索文字入力から結果表示まで100ms以内（インクリメンタルサーチ） |

## 3. 表示条件・バリデーション

| 条件 | 表示内容 |
|------|---------|
| 検索結果が0件の場合 | 「該当するドリンクが見つかりません」をカード一覧エリアに表示 |
| 「期間限定」タブで登録レシピが0件の場合 | 「まだ期間限定レシピが登録されていません。＋ボタンから追加しましょう」を表示 |
| システム組込みレシピの長押し | コンテキストメニューを表示しない（削除・編集不可） |

## 4. エラーケース

| エラー | 対応 |
|--------|------|
| レシピデータ取得失敗 | 「レシピを読み込めませんでした」表示＋再試行ボタン |
| ユーザーレシピ削除失敗 | 「削除に失敗しました。もう一度お試しください」トースト通知 |
